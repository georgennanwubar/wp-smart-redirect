Smart Redirect For Returning Users (v1.0)

Overview

This plugin redirects users to a defined URL if a specific cookie is detected in their browser. It includes GA4 event tracking, role-based exclusions, redirection logs, email alerts, and server-side tracking fallback using the GA4 Measurement Protocol.

Architecture

  • Frontend detection and redirection via JavaScript or PHP
  • Server-side event fallback via WP HTTP API
  • Settings stored using WordPress options API
  • Admin page registered with add_options_page

Hooks Used

  • init: To perform the redirect and log actions
  • admin_menu: To add the plugin settings page
  • admin_init: To register settings
  • wp_head: (conditionally) Injects gtag if enabled

Settings Fields

  • smart_applicable_urls – List of applicable URLs (newline separated)
  • smart_invert_url_logic – Inverts logic to exclude instead of include
  • smart_delete_data_on_uninstall – Deletes plugin settings/data on uninstall
  • smart_applicable_urls – List of URLs where the redirect should apply (newline separated)
  • smart_cookie_name – Cookie to check (default: smart_user_cookie)
  • smart_cookie_value – Expected value to trigger redirect
  • smart_redirect_url – Target redirect URL
  • smart_excluded_roles – User roles excluded from redirect
  • smart_redirect_count – Running count of total redirects
  • smart_redirect_logs – Logs of timestamp + IP
  • smart_email_threshold – Alert threshold for redirect count
  • smart_gtag_id – GA4 Measurement ID
  • smart_ga4_api_secret – GA4 Measurement Protocol API Secret
  • smart_enable_ga – Toggle for GA4 tracking (yes/no)

Redirection Logic

You can now invert URL logic to exclude the listed pages. Also, plugin cleanup is supported during uninstall.

The redirect logic only runs on the URLs you define in the settings. You can disable the default homepage redirect by removing it from the list.

On the init hook, the plugin checks if the current user is in an excluded role. If not, and the cookie condition is met, the following happens:

  1. Client-side redirect via JavaScript (with optional gtag)
  2. Server-side fallback event to GA4 if enabled
  3. Redirect logs recorded and stored in options
  4. Email sent to admin if count exceeds threshold

Admin Panel

Available under Settings → Smart Redirect. Offers complete control over all plugin functionality, including GA tracking and logging.

Logging

The plugin stores the last 1000 redirection events (timestamp + IP) in smart_redirect_logs option. You can export this log as a CSV from the admin panel.

Security & Best Practices

  • Uses WordPress native option and wp_remote_post APIs
  • Prevents redirect logic from running in is_admin() and DOING_AJAX contexts
  • Escapes output and input in settings fields
