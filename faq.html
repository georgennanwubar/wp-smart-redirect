Frequently Asked Questions

Q: What does this plugin do?

A: It redirects users to a specified URL (e.g., app dashboard) if a specific cookie exists and matches your criteria.

Q: When should I use this plugin?

A: When you want to automatically redirect logged-in or returning users to a custom page based on a cookie, e.g., after login or membership validation.

Q: Can I define the cookie name and value?

A: Yes. Go to Settings → Smart Redirect to configure the cookie name and expected value.

Q: What happens if the cookie is not found?

A: The user remains on the current page. No redirection is triggered.

Q: Can I exclude specific roles like Admin from being redirected?

A: Yes. The settings allow you to select one or more WordPress user roles to exclude from redirection logic.

Q: How can I track how many users have been redirected?

A: The plugin logs every redirect and shows the count in the settings panel. You can also export logs as a CSV file.

Q: What if a user blocks JavaScript?

A: The plugin includes a GA4 Measurement Protocol fallback that sends tracking events server-side even when JavaScript is blocked.

Q: What is the GA4 API Secret?

A: It is a key found in your Google Analytics Data Stream settings, used to send data to GA4 via server-side API calls.

Q: Is this GDPR compliant?

A: Yes. The plugin uses anonymous IP and cookie values only. You can control all tracking settings and disable them if needed.

Compatibility

Requires WordPress: 5.0+
Requires PHP: 7.0+

Q: Can I choose which pages this plugin runs on?

A: Yes. Go to the Smart Redirect settings and add each page or link (one per line) where the redirect should apply. You can leave out the homepage to disable default redirect behavior.

Q: Can I invert the page matching logic?

A: Yes. Enable 'Invert URL Logic' to redirect on all pages except the ones you enter.

Q: Can I delete plugin data on uninstall?

A: Yes. Enable the 'Delete Data On Uninstall' setting to wipe all logs and options when the plugin is removed.
